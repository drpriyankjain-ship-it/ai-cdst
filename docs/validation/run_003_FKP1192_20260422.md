## Recommended Next Steps (Priority Order)

1. **Thinking config** — Trial `thinking_config` with `thinking_budget=0` (disabled) or a low token budget on all 3.x calls. This is the single change most likely to make 3.x latency-competitive with 2.5-flash. Without it, 3.x is not viable for production.
2. **D2 `must_not_miss` regression** — Add explicit prompt instruction: "Check each differential entry against the must_not_miss list provided. Entries on that list must have `must_not_miss: true` regardless of probability ranking." Verify the must_not_miss.json list is being injected correctly into the D2 prompt.
3. **M2 prescribing** — Add explicit instruction: "For each problem, prescribe from the formulary where a drug is indicated and safe, even if the patient is being referred. Do not withhold prescribing solely because risk tier is HIGH or because MO review is required."
4. **D2 `referral_required`** — Add context to the D2 prompt clarifying that this is an SHC-HWC sub-centre setting; referral should be marked true for conditions requiring PHC/CHC-level diagnostics or specialist care.
5. **M2 `assessment` nested schema** — Investigate whether `response_json_schema` (vs `response_schema`) resolves nested object enforcement for Gemini 3.x.
