## Purpose of This Run

Three significant changes from run_003:

1. **Pro models removed.** TIER_BEST eliminated. D2 and M2 moved to TIER_STANDARD_CRITICAL (`gemini-3-flash-preview` → `gemini-2.5-pro` → `gemini-2.5-flash`). All other calls on TIER_STANDARD or TIER_FAST with 2.5 fallbacks.
2. **Thinking config removed.** `ThinkingConfig` stripped from all 9 calls and from the cascade helper. 2.5 models do not support `thinking_level` and were crashing on fallback in run_003b. No thinking configuration is now set — models use their defaults.
3. **M2 schema fixes (three bugs resolved):**
   - `assessment` field changed from untyped `{"type": "object"}` to a fully typed schema with properties per problem type. Previously caused Gemini constrained generation to emit whitespace floods and token loops on `icd10_code`.
   - `assessment` trimmed to functional fields only — removed `key_features_supporting`, `remaining_uncertainty`, `adherence_issue`, `probable_cause`. These were not consumed downstream and were driving 47k+ char outputs.
   - `validate_problem_list()` added as a post-generation sanitizer, analogous to `validate_differential()` in diagnosis_stage.py.
